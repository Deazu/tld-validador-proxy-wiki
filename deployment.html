Infraestructura template.yaml

Este documento es el manual del plano de construcción de la Lambda tld-validador-proxy, en el mismo espíritu que samconfig.toml documenta los parámetros de despliegue: aquí definimos qué construye CloudFormation/SAM (función, VPC, EFS, IAM, DynamoDB de telemetría) para que el equipo pueda versionar infraestructura y aplicación en un solo repositorio.

Rol del template.yaml

El archivo es el contrato declarativo entre ingeniería y AWS: describe la Lambda con VpcConfig alineada a los stacks satélite, el montaje FileSystemConfigs en /mnt/tld-llaves para CA, CRL y material PEM operativo, variables de entorno, permisos para leer las tablas tld-validador-canal*, usar KMS según el origen del motor criptográfico, y escribir en la tabla tld-validador-proxy-telemetry. Cualquier desviación respecto a P2M / cuenta-nombre / alias debe quedar explícita en parámetros o en ADR.

Recursos que debe cubrir el plano

  • AWS::Serverless::Function (o equivalente) con runtime Node.js, handler, código en lambdas/proxy/, timeout/memoria acordes al cold start EFS + TLS.
  • VpcConfig: mismos SecurityGroupIds y SubnetIds que el servicio hermano en cada ambiente (valores inyectados vía samconfig.toml).
  • FileSystemConfigs: ARN del access point de EFS y LocalMountPath: /mnt/tld-llaves.
  • IAM: lectura (y si aplica escritura acotada) sobre tablas tld-validador-canal, tld-validador-canal-llave, tld-validador-canal-operacion y similares; kms:Encrypt / kms:Decrypt sobre la clave indicada; dynamodb:PutItem (o document API) sobre la tabla de telemetría; permisos EFS para montaje.
  • API privada (opcional en la plantilla mínima): si el proxy se expone vía API Gateway, replicar EndpointConfiguration: PRIVATE y VPCEndpointIds como en los APIs existentes.

Tabla tld-validador-proxy-telemetry

Tabla dedicada a observabilidad técnica (fases TLS, CRL, cifrado, latencias). No sustituye auditoría de negocio.

Claves sugeridas: PK correlationId, SK timestamp (epoch ms). Atributos: phase, outcome, durationMs, expireAt (TTL), etc.

Relación con samconfig.toml

Los parameter_overrides del perfil SAM deben llenar los parámetros declarados abajo (VPCe solo si usas API privada con resource policy, subnets, SG, AccessPointId, KmsKeyArn, rutas CA/CRL, nombre de tabla de telemetría si lo externalizas). La matriz de IDs por ambiente está en Manual samconfig.toml.

Plantilla template.yaml propuesta (copiar)

Borrador de ingeniería: validar ARNs, nombres de tablas y políticas con el equipo de plataforma antes del primer sam deploy. Ajustar CodeUri, Handler y recursos opcionales (API, alarmas, SSM) según el repo real.

template.yaml 
AWSTemplateFormatVersion: '2010-09-09'
Transform: AWS::Serverless-2016-10-31
Description: >
  tld-validador-proxy — plano SAM (Lambda + VPC + EFS + telemetría DynamoDB + IAM canal/KMS).
  Alinear parameter_overrides con network.html / samconfig.toml.

Globals:
  Function:
    Timeout: 60
    MemorySize: 512
    Runtime: nodejs20.x
    Architectures:
      - x86_64

Parameters:
  DeployEnvironment:
    Type: String
    Description: Etiqueta de despliegue (dev, sandbox, qa, prod, etc.)

  SecurityGroup:
    Type: String
    Description: Security Group de la Lambda en la VPC compartida

  Subnet1:
    Type: String
  Subnet2:
    Type: String
  Subnet3:
    Type: String

  AccessPointId:
    Type: String
    Description: ID del EFS Access Point (fsap-...)

  KmsKeyArn:
    Type: String
    Description: ARN completo de la CMK/MRK para Encrypt/Decrypt vía operacionesPaquete

  RutaArchivoCa:
    Type: String
    Default: /mnt/tld-llaves/tld-ca/caroot.cer
  RutaArchivoCaSub:
    Type: String
    Default: /mnt/tld-llaves/tld-ca/casub.cer
  RutaArchivoCrl:
    Type: String
    Default: /mnt/tld-llaves/tld-crl/crltelered.crl

Resources:
  ProxyTelemetryTable:
    Type: AWS::DynamoDB::Table
    Properties:
      TableName: tld-validador-proxy-telemetry
      BillingMode: PAY_PER_REQUEST
      AttributeDefinitions:
        - AttributeName: correlationId
          AttributeType: S
        - AttributeName: timestamp
          AttributeType: N
      KeySchema:
        - AttributeName: correlationId
          KeyType: HASH
        - AttributeName: timestamp
          KeyType: RANGE
      TimeToLiveSpecification:
        AttributeName: expireAt
        Enabled: true
      Tags:
        - Key: Project
          Value: tld-validador-proxy
        - Key: Backup
          Value: 'yes'

  ValidatorProxyFunction:
    Type: AWS::Serverless::Function
    Properties:
      FunctionName: !Sub 'tld-validador-proxy-${DeployEnvironment}'
      CodeUri: lambdas/proxy/
      Handler: app.handler
      VpcConfig:
        SecurityGroupIds:
          - !Ref SecurityGroup
        SubnetIds:
          - !Ref Subnet1
          - !Ref Subnet2
          - !Ref Subnet3
      FileSystemConfigs:
        - Arn: !Sub 'arn:aws:elasticfilesystem:${AWS::Region}:${AWS::AccountId}:access-point/${AccessPointId}'
          LocalMountPath: /mnt/tld-llaves
      Environment:
        Variables:
          DeployEnvironment: !Ref DeployEnvironment
          PROXY_TELEMETRY_TABLE: !Ref ProxyTelemetryTable
          LIB_RUTA_ARCHIVO_CA: !Ref RutaArchivoCa
          LIB_RUTA_ARCHIVO_CA_SUB: !Ref RutaArchivoCaSub
          LIB_RUTA_ARCHIVO_CRL: !Ref RutaArchivoCrl
          PRINT_LOGS: 'off'
      Policies:
        - Statement:
            - Sid: DynamoReadCanalFamily
              Effect: Allow
              Action:
                - dynamodb:GetItem
                - dynamodb:Query
                - dynamodb:Scan
                - dynamodb:BatchGetItem
              Resource:
                - !Sub 'arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal'
                - !Sub 'arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal/index/*'
                - !Sub 'arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-llave'
                - !Sub 'arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion'
                - !Sub 'arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion/index/*'
                - !Sub 'arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-auth-token'
            - Sid: DynamoWriteTelemetry
              Effect: Allow
              Action:
                - dynamodb:PutItem
              Resource: !GetAtt ProxyTelemetryTable.Arn
            - Sid: KmsForCryptoPackage
              Effect: Allow
              Action:
                - kms:Decrypt
                - kms:Encrypt
              Resource: !Ref KmsKeyArn
            - Sid: EfsMountAccessPoint
              Effect: Allow
              Action:
                - elasticfilesystem:ClientMount
                - elasticfilesystem:ClientRead
              Resource: !Sub 'arn:aws:elasticfilesystem:${AWS::Region}:${AWS::AccountId}:access-point/${AccessPointId}'

Outputs:
  ValidatorProxyFunctionArn:
    Description: ARN de la función proxy
    Value: !GetAtt ValidatorProxyFunction.Arn
  ProxyTelemetryTableName:
    Description: Tabla de telemetría técnica
    Value: !Ref ProxyTelemetryTable
