Criterios de diseño — referencia estructural

  • Orden SAM: Se conserva el orden del molde de casa: bloque Parameters (VPC, subnets, SG, timeout, EFS, rutas CA/CRL, tablas), Globals.Function con VpcConfig y variables de entorno compartidas, Conditions para recursos globales por región, y Resources.
  • Red y PKI: Los bloques VpcConfig (en Globals) y FileSystemConfigs (por función, con LocalMountPath: /mnt/tld-llaves) se modelan como en el stack de notificaciones, para que operaciones reconozcan el patrón de montaje EFS y certificados.
  • API privada: Se mantiene el esquema AWS::Serverless::Api con EndpointConfiguration: PRIVATE y VPCEndpointIds, análogo a NotificacionApi (política de recurso con deny fuera del VPCE).
  • Resiliencia: A diferencia del original, la cola de trabajo del proxy no queda aislada: lleva RedrivePolicy hacia una DLQ dedicada y se elimina el antipatrón de política SQS con Principal: '*' sobre recursos amplios; el envío queda acotado al rol de la Lambda API (y, si aplica, políticas explícitas por ARN).

Por qué DLQ y por qué ajustar VisibilityTimeout

En el diseño previo analizado, la cola principal no tenía DLQ configurada en plantilla: los mensajes podían reentregarse indefinidamente cuando el procesador registraba la excepción pero no actualizaba contadores ni movía el mensaje a un destino de análisis. Eso dificulta distinguir “cola profunda por incidente” de “veneno silencioso” y retrasa la respuesta operativa.

La ProxyDeadLetterQueue (tld-validador-proxy-resilience-dlq) cumple tres funciones: (1) preservar el payload tras agotar maxReceiveCount en la cola principal, para inspección y reproceso controlado; (2) disparar alarmas CloudWatch sobre profundidad de DLQ; (3) evitar la pérdida aparente de notificaciones cuando el worker falla de forma sostenida.

El VisibilityTimeout de la cola principal debe ser estrictamente mayor que el tiempo máximo que la Lambda worker puede tardar en procesar un mensaje (incluyendo llamadas HTTP al validador y backoff interno acotado). Si el timeout de visibilidad es igual o menor que el de la función, SQS puede volver a entregar el mensaje mientras la invocación sigue viva, generando doble procesamiento y presión duplicada sobre el banco. La guía habitual de AWS es fijar la visibilidad en al menos seis veces el timeout de la Lambda consumidora (margen para cold start, extensiones y DeleteMessage). En el fragmento propuesto se usa 360 segundos frente a Timeout: 60 en Globals como punto de partida; el equipo debe validarlo contra el peor caso real del worker.

Nombres de recursos (convención proxy)

Recurso lógico / físico Nombre / patrón
Cola de resiliencia (trabajo asíncrono) tld-validador-proxy-resilience-queue (QueueName)
DLQ tld-validador-proxy-resilience-dlq
Lambda API (handler síncrono + encolado 202) tld-validador-proxy-api (ajustar sufijo por ambiente si política de nombres lo exige)
Lambda worker (consumidor SQS) tld-validador-proxy-resilience-worker
API Gateway (privada) Nombre de stack / etapa alineados a DeployEnvironment (mismo criterio que NotificacionApi)

template.yaml propuesto (fragmento de stack)

Borrador de ingeniería: validar con sam validate, permisos de tablas reales (tld-validador-canal*), KMS y rutas EFS por cuenta. Los Policies sustituyen políticas CRUD genéricas donde el proxy solo necesita lectura de canal u operación; amplía acciones si el código lo exige. La cola policy usa el rol generado por SAM (ProxyApiFunctionRole).

template.yaml — tld-validador-proxy (molde SAM casa + resiliencia) 
AWSTemplateFormatVersion: "2010-09-09"
Transform: AWS::Serverless-2016-10-31
Description: >
  tld-validador-proxy
  API validador + resiliencia SQS (DLQ) + worker; mismo esqueleto SAM que tld-notificacion.

Parameters:

  DeployEnvironment:
    Type: String
    AllowedValues:
      - dev
      - sandbox
      - qa
      - tlrd-highway
    Default: dev
    Description: Nombre del ambiente de despliegue.

  VPCe:
    Type: String
    AllowedPattern: "^(vpce-).*"
    Description: VPCE id para API Gateway privado.

  SecurityGroup:
    Type: AWS::EC2::SecurityGroup::Id
    Description: Security group id para Lambdas.

  Subnet1:
    Type: AWS::EC2::Subnet::Id
    Description: Primera SubnetId para Lambdas.

  Subnet2:
    Type: AWS::EC2::Subnet::Id
    Description: Segunda SubnetId para Lambdas.

  Subnet3:
    Type: AWS::EC2::Subnet::Id
    Description: Tercera SubnetId para Lambdas.

  Timeout:
    Type: Number
    Default: 60
    Description: Timeout Lambda (segundos); usar con VisibilityTimeout de cola ≥ 6× este valor.

  ProxyTelemetryTableName:
    Type: String
    Default: "tld-validador-proxy-telemetry"

  ResilienceQueueVisibilityTimeout:
    Type: Number
    Default: 360
    Description: Segundos de visibilidad SQS (recomendado ≥ 6 × Timeout).

  ResilienceMaxReceiveCount:
    Type: Number
    Default: 5
    Description: Recepciones antes de DLQ; alinear con política Opt-in y telemetría.

  AccessPointId:
    Type: String
    Description: AccessPoint EFS para llaves y PKI.
    Default: fsap-069f5ed286aca3a98

  EFSResourceId:
    Type: String
    Description: File System Id EFS.
    Default: fs-0e4079df1cdb063b3

  RutaArchivoCa:
    Type: String
    Default: /mnt/tld-llaves/tld-ca/caroot.cer
  RutaArchivoCaSub:
    Type: String
    Default: /mnt/tld-llaves/tld-ca/casub.cer
  RutaArchivoCrl:
    Type: String
    Default: /mnt/tld-llaves/tld-crl/crltelered.crl

  KmsKeyArn:
    Type: String
    Description: ARN CMK/MRK para cifrado de aplicación según operacionesPaquete.

Globals:
  Function:
    Timeout: !Ref Timeout
    MemorySize: 512
    VpcConfig:
      SecurityGroupIds:
        - !Ref SecurityGroup
      SubnetIds:
        - !Ref Subnet1
        - !Ref Subnet2
        - !Ref Subnet3
    Environment:
      Variables:
        AWS_NODEJS_CONNECTION_REUSE_ENABLED: "1"
        PRINT_LOGS: "on"
        LIB_RUTA_ARCHIVO_CA: !Ref RutaArchivoCa
        LIB_RUTA_ARCHIVO_CA_SUB: !Ref RutaArchivoCaSub
        LIB_RUTA_ARCHIVO_CRL: !Ref RutaArchivoCrl

Conditions:
  CreateGlobalResources: !Equals
    - !Ref "AWS::Region"
    - "us-east-1"
  CreateGlobalResourcesDR: !Not
    - !Equals
      - !Ref "AWS::Region"
      - "us-east-1"

Resources:

  ValidadorProxyApi:
    Type: AWS::Serverless::Api
    Properties:
      Name: !Sub "${AWS::StackName}-proxy"
      StageName: !Ref DeployEnvironment
      EndpointConfiguration:
        Type: PRIVATE
        VPCEndpointIds:
          - !Ref VPCe
      Auth:
        ResourcePolicy:
          CustomStatements:
            - Effect: Allow
              Principal: "*"
              Action: "execute-api:Invoke"
              Resource: ["execute-api:/*/*/*"]
            - Effect: Deny
              Principal: "*"
              Action: "execute-api:Invoke"
              Resource: ["execute-api:/*/*/*"]
              Condition:
                StringNotEquals:
                  aws:SourceVpce: !Ref VPCe
      Cors:
        AllowMethods: "'DELETE,GET,HEAD,OPTIONS,PATCH,POST,PUT'"
        AllowHeaders: "'Content-Type,Authorization,X-Amz-Date,X-Api-Key,X-Amz-Security-Token'"
        AllowOrigin: "'*'"

  ProxyDeadLetterQueue:
    Type: AWS::SQS::Queue
    Properties:
      QueueName: tld-validador-proxy-resilience-dlq
      MessageRetentionPeriod: 1209600
      Tags:
        - Key: Project
          Value: tld-validador-proxy

  ProxyResilienceQueue:
    Type: AWS::SQS::Queue
    Properties:
      QueueName: tld-validador-proxy-resilience-queue
      VisibilityTimeout: !Ref ResilienceQueueVisibilityTimeout
      ReceiveMessageWaitTimeSeconds: 20
      MessageRetentionPeriod: 1209600
      RedrivePolicy:
        deadLetterTargetArn: !GetAtt ProxyDeadLetterQueue.Arn
        maxReceiveCount: !Ref ResilienceMaxReceiveCount
      Tags:
        - Key: Project
          Value: tld-validador-proxy

  ProxyResilienceQueuePolicy:
    Type: AWS::SQS::QueuePolicy
    DependsOn:
      - ProxyApiFunction
    Properties:
      Queues:
        - !Ref ProxyResilienceQueue
      PolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Sid: AllowSendFromProxyApiRoleOnly
            Effect: Allow
            Principal:
              AWS: !GetAtt ProxyApiFunctionRole.Arn
            Action: "sqs:SendMessage"
            Resource: !GetAtt ProxyResilienceQueue.Arn

  ProxyTelemetryTable:
    Type: AWS::DynamoDB::Table
    Condition: CreateGlobalResources
    Properties:
      TableName: !Ref ProxyTelemetryTableName
      BillingMode: PAY_PER_REQUEST
      AttributeDefinitions:
        - AttributeName: correlationId
          AttributeType: S
        - AttributeName: timestamp
          AttributeType: N
      KeySchema:
        - AttributeName: correlationId
          KeyType: HASH
        - AttributeName: timestamp
          KeyType: RANGE
      TimeToLiveSpecification:
        AttributeName: expireAt
        Enabled: true
      PointInTimeRecoverySpecification:
        PointInTimeRecoveryEnabled: true
      Tags:
        - Key: Project
          Value: tld-validador-proxy

  ProxyApiFunction:
    Type: AWS::Serverless::Function
    Properties:
      FunctionName: tld-validador-proxy-api
      CodeUri: lambdas/proxy/
      Handler: app.handler
      Runtime: nodejs20.x
      FileSystemConfigs:
        - Arn: !Sub
            - "arn:aws:elasticfilesystem:${AWS::Region}:${AWS::AccountId}:access-point/${AccessPointId}"
            - AccessPointId: !Ref AccessPointId
          LocalMountPath: /mnt/tld-llaves
      Environment:
        Variables:
          PROXY_TELEMETRY_TABLE: !Ref ProxyTelemetryTableName
          PROXY_RESILIENCE_QUEUE_URL: !Ref ProxyResilienceQueue
      Policies:
        - Statement:
            - Sid: DynamoReadCanalFamily
              Effect: Allow
              Action:
                - dynamodb:GetItem
                - dynamodb:Query
                - dynamodb:BatchGetItem
              Resource:
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal/index/*"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-llave"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion/index/*"
            - Sid: DynamoWriteTelemetry
              Effect: Allow
              Action:
                - dynamodb:PutItem
              Resource: !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/${ProxyTelemetryTableName}"
            - Sid: SqsSendResilienceQueue
              Effect: Allow
              Action:
                - sqs:SendMessage
                - sqs:GetQueueUrl
              Resource: !GetAtt ProxyResilienceQueue.Arn
            - Sid: KmsCrypto
              Effect: Allow
              Action:
                - kms:Decrypt
                - kms:Encrypt
              Resource: !Ref KmsKeyArn
        - EFSWriteAccessPolicy:
            FileSystem: !Ref EFSResourceId
            AccessPoint: !Ref AccessPointId
      Events:
        ProxyHttpApi:
          Type: Api
          Properties:
            RestApiId: !Ref ValidadorProxyApi
            Path: /proxy
            Method: post

  ProxyWorkerFunction:
    Type: AWS::Serverless::Function
    Properties:
      FunctionName: tld-validador-proxy-resilience-worker
      CodeUri: lambdas/proxy-worker/
      Handler: worker.handler
      Runtime: nodejs20.x
      FileSystemConfigs:
        - Arn: !Sub
            - "arn:aws:elasticfilesystem:${AWS::Region}:${AWS::AccountId}:access-point/${AccessPointId}"
            - AccessPointId: !Ref AccessPointId
          LocalMountPath: /mnt/tld-llaves
      Environment:
        Variables:
          PROXY_TELEMETRY_TABLE: !Ref ProxyTelemetryTableName
          PROXY_RESILIENCE_QUEUE_URL: !Ref ProxyResilienceQueue
      Policies:
        - Statement:
            - Sid: DynamoReadCanalFamily
              Effect: Allow
              Action:
                - dynamodb:GetItem
                - dynamodb:Query
                - dynamodb:BatchGetItem
              Resource:
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal/index/*"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-llave"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion/index/*"
            - Sid: DynamoWriteTelemetry
              Effect: Allow
              Action:
                - dynamodb:PutItem
              Resource: !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/${ProxyTelemetryTableName}"
            - Sid: SqsConsumeResilienceQueue
              Effect: Allow
              Action:
                - sqs:ReceiveMessage
                - sqs:DeleteMessage
                - sqs:GetQueueAttributes
                - sqs:ChangeMessageVisibility
              Resource: !GetAtt ProxyResilienceQueue.Arn
            - Sid: SqsSendForRequeueOptional
              Effect: Allow
              Action:
                - sqs:SendMessage
              Resource: !GetAtt ProxyResilienceQueue.Arn
            - Sid: KmsCrypto
              Effect: Allow
              Action:
                - kms:Decrypt
                - kms:Encrypt
              Resource: !Ref KmsKeyArn
        - EFSWriteAccessPolicy:
            FileSystem: !Ref EFSResourceId
            AccessPoint: !Ref AccessPointId
      Events:
        ResilienceQueueEvent:
          Type: SQS
          Properties:
            Queue: !GetAtt ProxyResilienceQueue.Arn
            BatchSize: 5
            FunctionResponseTypes:
              - ReportBatchItemFailures

Outputs:
  ProxyResilienceQueueUrl:
    Description: URL cola de trabajo asíncrono
    Value: !Ref ProxyResilienceQueue
  ProxyDeadLetterQueueUrl:
    Description: URL DLQ para análisis y reproceso
    Value: !Ref ProxyDeadLetterQueue
  ProxyApiFunctionArn:
    Description: ARN Lambda API
    Value: !GetAtt ProxyApiFunction.Arn
  ProxyWorkerFunctionArn:
    Description: ARN Lambda worker SQS
    Value: !GetAtt ProxyWorkerFunction.Arn
IAM granular: Las Lambdas no usan DynamoDBCrudPolicy genérico salvo necesidad; las acciones SQS del worker listan solo recepción, borrado, visibilidad y (opcional) reenvío a la misma cola. La política de cola reemplaza el Principal: '*' / Resource: '*' del stack legado: solo el rol ProxyApiFunctionRole puede SendMessage a ProxyResilienceQueue a nivel de cola (defensa en profundidad junto al IAM de la función).
