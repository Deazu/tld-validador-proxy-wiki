Infraestructura SAM — template.yaml del Proxy

Plano AWS SAM del tld-validador-proxy con la misma estructura de casa que tld-notificacion (Parameters → Globals → Conditions → Resources), pero sin API Gateway: el borde hacia el proxy es invocación directa de Lambda (lambda:InvokeFunction) desde satélites en la misma cuenta (u orígenes autorizados vía IAM). Se conservan VPC, EFS y colas SQS con DLQ para la Capa 2.

Criterios de diseño — referencia estructural

  • Orden SAM: Parameters (VPC, subnets, SG, timeout, EFS, rutas CA/CRL, tablas, colas), Globals.Function con VpcConfig y variables compartidas, Conditions, Resources.
  • Red y PKI: VpcConfig en Globals y FileSystemConfigs por función con LocalMountPath: /mnt/tld-llaves, igual que en el stack de notificaciones.
  • Sin AWS::Serverless::Api: no hay Events: Api ni VPCEndpointIds para Gateway; la conectividad interna es SDK → Lambda.
  • Resiliencia: ProxyResilienceQueue con RedrivePolicy hacia ProxyDeadLetterQueue; política de cola sin Principal: '*' ampliado — solo el rol de la Lambda proxy puede SendMessage donde aplique.

Invocación directa — permiso lambda:InvokeFunction

Cada satélite que deba llamar al proxy (p. ej. inactividad, alias, p2m) debe llevar en su rol de ejecución una sentencia IAM explícita que permita lambda:InvokeFunction sobre el ARN de la función tld-validador-proxy-api (o el nombre que despliegue el stack). No se expone URL HTTP interna del proxy en este modelo.

El timeout efectivo que percibe el llamador lo fija el cliente AWS SDK en el satélite (LambdaClient con configuración de client timeout / intentos de red), además del Timeout configurado en la Lambda del proxy. El equipo debe dimensionar ambos de forma coherente con la resiliencia controlada (Capa 1) documentada en Estrategia y Capa 1 y Diccionario de datos.

Contrato de payload y respuesta (incl. semántica 202 lógica en el cuerpo de retorno): Flujo General.

Por qué DLQ y por qué ajustar VisibilityTimeout

En el diseño previo analizado, la cola principal no tenía DLQ configurada en plantilla: los mensajes podían reentregarse indefinidamente cuando el procesador registraba la excepción pero no actualizaba contadores ni movía el mensaje a un destino de análisis.

La ProxyDeadLetterQueue (tld-validador-proxy-resilience-dlq) preserva el payload tras agotar maxReceiveCount, habilita alarmas y reduce la pérdida aparente de notificaciones cuando el worker falla de forma sostenida.

El VisibilityTimeout de la cola principal debe ser mayor que el tiempo máximo de procesamiento del worker (HTTP al validador + backoff). La guía habitual es al menos seis veces el timeout de la Lambda consumidora. En el fragmento se usa 360 s frente a Timeout: 60 en Globals como punto de partida.

Nombres de recursos (convención proxy)

Recurso lógico / físico Nombre / patrón
Cola de resiliencia tld-validador-proxy-resilience-queue
DLQ tld-validador-proxy-resilience-dlq
Lambda principal (invocable por SDK) tld-validador-proxy-api — destino de lambda:InvokeFunction
Lambda worker (SQS) tld-validador-proxy-resilience-worker
Borde de acceso Invocación directa (AWS SDK / consola / EventBridge según diseño); no API Gateway en esta plantilla.

template.yaml propuesto (fragmento de stack)

Borrador: sam validate. Añade en el stack del llamador la política IAM con lambda:InvokeFunction al ARN exportado (o usa Fn::ImportValue si publicas outputs entre stacks). Sin AWS::Serverless::Api ni Events: Api.

template.yaml — tld-validador-proxy (Lambda + SQS DLQ + EFS) 
AWSTemplateFormatVersion: "2010-09-09"
Transform: AWS::Serverless-2016-10-31
Description: >
  tld-validador-proxy
  Lambda invocable por SDK + resiliencia SQS (DLQ) + worker; molde SAM alineado a tld-notificacion sin API Gateway.

Parameters:

  DeployEnvironment:
    Type: String
    AllowedValues:
      - dev
      - sandbox
      - qa
      - tlrd-highway
    Default: dev
    Description: Nombre del ambiente de despliegue.

  SecurityGroup:
    Type: AWS::EC2::SecurityGroup::Id
    Description: Security group id para Lambdas.

  Subnet1:
    Type: AWS::EC2::Subnet::Id
    Description: Primera SubnetId para Lambdas.

  Subnet2:
    Type: AWS::EC2::Subnet::Id
    Description: Segunda SubnetId para Lambdas.

  Subnet3:
    Type: AWS::EC2::Subnet::Id
    Description: Tercera SubnetId para Lambdas.

  Timeout:
    Type: Number
    Default: 60
    Description: Timeout Lambda (segundos); alinear con cliente SDK del llamador y VisibilityTimeout SQS ≥ 6× worker.

  ProxyTelemetryTableName:
    Type: String
    Default: "tld-validador-proxy-telemetry"

  ResilienceQueueVisibilityTimeout:
    Type: Number
    Default: 360
    Description: Visibilidad SQS (recomendado ≥ 6 × Timeout del worker).

  ResilienceMaxReceiveCount:
    Type: Number
    Default: 5

  AccessPointId:
    Type: String
    Description: AccessPoint EFS para llaves y PKI.
    Default: fsap-069f5ed286aca3a98

  EFSResourceId:
    Type: String
    Description: File System Id EFS.
    Default: fs-0e4079df1cdb063b3

  RutaArchivoCa:
    Type: String
    Default: /mnt/tld-llaves/tld-ca/caroot.cer
  RutaArchivoCaSub:
    Type: String
    Default: /mnt/tld-llaves/tld-ca/casub.cer
  RutaArchivoCrl:
    Type: String
    Default: /mnt/tld-llaves/tld-crl/crltelered.crl

  KmsKeyArn:
    Type: String
    Description: ARN CMK/MRK para cifrado de aplicación según operacionesPaquete.

Globals:
  Function:
    Timeout: !Ref Timeout
    MemorySize: 512
    VpcConfig:
      SecurityGroupIds:
        - !Ref SecurityGroup
      SubnetIds:
        - !Ref Subnet1
        - !Ref Subnet2
        - !Ref Subnet3
    Environment:
      Variables:
        AWS_NODEJS_CONNECTION_REUSE_ENABLED: "1"
        PRINT_LOGS: "on"
        LIB_RUTA_ARCHIVO_CA: !Ref RutaArchivoCa
        LIB_RUTA_ARCHIVO_CA_SUB: !Ref RutaArchivoCaSub
        LIB_RUTA_ARCHIVO_CRL: !Ref RutaArchivoCrl

Conditions:
  CreateGlobalResources: !Equals
    - !Ref "AWS::Region"
    - "us-east-1"
  CreateGlobalResourcesDR: !Not
    - !Equals
      - !Ref "AWS::Region"
      - "us-east-1"

Resources:

  ProxyDeadLetterQueue:
    Type: AWS::SQS::Queue
    Properties:
      QueueName: tld-validador-proxy-resilience-dlq
      MessageRetentionPeriod: 1209600
      Tags:
        - Key: Project
          Value: tld-validador-proxy

  ProxyResilienceQueue:
    Type: AWS::SQS::Queue
    Properties:
      QueueName: tld-validador-proxy-resilience-queue
      VisibilityTimeout: !Ref ResilienceQueueVisibilityTimeout
      ReceiveMessageWaitTimeSeconds: 20
      MessageRetentionPeriod: 1209600
      RedrivePolicy:
        deadLetterTargetArn: !GetAtt ProxyDeadLetterQueue.Arn
        maxReceiveCount: !Ref ResilienceMaxReceiveCount
      Tags:
        - Key: Project
          Value: tld-validador-proxy

  ProxyResilienceQueuePolicy:
    Type: AWS::SQS::QueuePolicy
    DependsOn:
      - ProxyApiFunction
    Properties:
      Queues:
        - !Ref ProxyResilienceQueue
      PolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Sid: AllowSendFromProxyApiRoleOnly
            Effect: Allow
            Principal:
              AWS: !GetAtt ProxyApiFunctionRole.Arn
            Action: "sqs:SendMessage"
            Resource: !GetAtt ProxyResilienceQueue.Arn

  ProxyTelemetryTable:
    Type: AWS::DynamoDB::Table
    Condition: CreateGlobalResources
    Properties:
      TableName: !Ref ProxyTelemetryTableName
      BillingMode: PAY_PER_REQUEST
      AttributeDefinitions:
        - AttributeName: correlationId
          AttributeType: S
        - AttributeName: timestamp
          AttributeType: N
      KeySchema:
        - AttributeName: correlationId
          KeyType: HASH
        - AttributeName: timestamp
          KeyType: RANGE
      TimeToLiveSpecification:
        AttributeName: expireAt
        Enabled: true
      PointInTimeRecoverySpecification:
        PointInTimeRecoveryEnabled: true
      Tags:
        - Key: Project
          Value: tld-validador-proxy

  ProxyApiFunction:
    Type: AWS::Serverless::Function
    Properties:
      FunctionName: tld-validador-proxy-api
      CodeUri: lambdas/proxy/
      Handler: app.handler
      Runtime: nodejs20.x
      FileSystemConfigs:
        - Arn: !Sub
            - "arn:aws:elasticfilesystem:${AWS::Region}:${AWS::AccountId}:access-point/${AccessPointId}"
            - AccessPointId: !Ref AccessPointId
          LocalMountPath: /mnt/tld-llaves
      Environment:
        Variables:
          PROXY_TELEMETRY_TABLE: !Ref ProxyTelemetryTableName
          PROXY_RESILIENCE_QUEUE_URL: !Ref ProxyResilienceQueue
          DEPLOY_ENVIRONMENT: !Ref DeployEnvironment
      Policies:
        - Statement:
            - Sid: DynamoReadCanalFamily
              Effect: Allow
              Action:
                - dynamodb:GetItem
                - dynamodb:Query
                - dynamodb:BatchGetItem
              Resource:
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal/index/*"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-llave"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion/index/*"
            - Sid: DynamoWriteTelemetry
              Effect: Allow
              Action:
                - dynamodb:PutItem
              Resource: !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/${ProxyTelemetryTableName}"
            - Sid: SqsSendResilienceQueue
              Effect: Allow
              Action:
                - sqs:SendMessage
                - sqs:GetQueueUrl
              Resource: !GetAtt ProxyResilienceQueue.Arn
            - Sid: KmsCrypto
              Effect: Allow
              Action:
                - kms:Decrypt
                - kms:Encrypt
              Resource: !Ref KmsKeyArn
        - EFSWriteAccessPolicy:
            FileSystem: !Ref EFSResourceId
            AccessPoint: !Ref AccessPointId

  ProxyWorkerFunction:
    Type: AWS::Serverless::Function
    Properties:
      FunctionName: tld-validador-proxy-resilience-worker
      CodeUri: lambdas/proxy-worker/
      Handler: worker.handler
      Runtime: nodejs20.x
      FileSystemConfigs:
        - Arn: !Sub
            - "arn:aws:elasticfilesystem:${AWS::Region}:${AWS::AccountId}:access-point/${AccessPointId}"
            - AccessPointId: !Ref AccessPointId
          LocalMountPath: /mnt/tld-llaves
      Environment:
        Variables:
          PROXY_TELEMETRY_TABLE: !Ref ProxyTelemetryTableName
          PROXY_RESILIENCE_QUEUE_URL: !Ref ProxyResilienceQueue
      Policies:
        - Statement:
            - Sid: DynamoReadCanalFamily
              Effect: Allow
              Action:
                - dynamodb:GetItem
                - dynamodb:Query
                - dynamodb:BatchGetItem
              Resource:
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal/index/*"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-llave"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion"
                - !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/tld-validador-canal-operacion/index/*"
            - Sid: DynamoWriteTelemetry
              Effect: Allow
              Action:
                - dynamodb:PutItem
              Resource: !Sub "arn:aws:dynamodb:${AWS::Region}:${AWS::AccountId}:table/${ProxyTelemetryTableName}"
            - Sid: SqsConsumeResilienceQueue
              Effect: Allow
              Action:
                - sqs:ReceiveMessage
                - sqs:DeleteMessage
                - sqs:GetQueueAttributes
                - sqs:ChangeMessageVisibility
              Resource: !GetAtt ProxyResilienceQueue.Arn
            - Sid: SqsSendForRequeueOptional
              Effect: Allow
              Action:
                - sqs:SendMessage
              Resource: !GetAtt ProxyResilienceQueue.Arn
            - Sid: KmsCrypto
              Effect: Allow
              Action:
                - kms:Decrypt
                - kms:Encrypt
              Resource: !Ref KmsKeyArn
        - EFSWriteAccessPolicy:
            FileSystem: !Ref EFSResourceId
            AccessPoint: !Ref AccessPointId
      Events:
        ResilienceQueueEvent:
          Type: SQS
          Properties:
            Queue: !GetAtt ProxyResilienceQueue.Arn
            BatchSize: 5
            FunctionResponseTypes:
              - ReportBatchItemFailures

Outputs:
  ProxyApiFunctionArn:
    Description: ARN Lambda proxy — usar en IAM del llamador (lambda:InvokeFunction)
    Value: !GetAtt ProxyApiFunction.Arn
  ProxyApiFunctionName:
    Description: Nombre función — Invoke(FunctionName)
    Value: !Ref ProxyApiFunction
  ProxyResilienceQueueUrl:
    Description: URL cola trabajo asíncrono
    Value: !Ref ProxyResilienceQueue
  ProxyDeadLetterQueueUrl:
    Description: URL DLQ
    Value: !Ref ProxyDeadLetterQueue
  ProxyWorkerFunctionArn:
    Description: ARN Lambda worker SQS
    Value: !GetAtt ProxyWorkerFunction.Arn
IAM del llamador (fuera de este snippet): el rol de la Lambda satélite debe incluir lambda:InvokeFunction sobre !GetAtt ProxyApiFunction.Arn (o el ARN estable del entorno). Así se evita API Gateway y se mantiene menor privilegio frente a políticas SQS con Principal '*' del legado.
